Architectural Strategy: Shared vs. Dedicated

Choosing between shared and dedicated gateways is a strategic decision involving trade-offs between blast radius, operational overhead, and cost.

Option A: Shared Gateway

A single Ingress Gateway deployment handles traffic for the entire cluster or multiple namespaces.

  • Cost Efficiency: One LoadBalancer, minimal idle resources.
  • Centralized Operations: Certificates and common policies managed in one place.
  • Risk: Noisy neighbors can impact all apps; shared config increases the blast radius of misconfigurations.

Option B: Dedicated Gateway

Each application or team deploys their own Ingress Gateway instance.

  • Maximum Isolation: Resource exhaustion or failure in one gateway doesn't affect others.
  • Customized Security: Tailor Gateway configuration and auth policies precisely to the app's needs.
  • Cost: Higher cloud bill due to multiple LoadBalancers and redundant gateway pods.
